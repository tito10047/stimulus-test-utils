# Cookbook

Short, copy-pasteable recipes for the scenarios that come up in every Stimulus project. Each page is self-contained: one problem, one solution, one complete test.

## Recipes

- [Testing values](./values.md) — defaults, overrides, programmatic changes, value changed callbacks.
- [Testing targets & classes](./targets-and-classes.md) — `hasXTarget`, `xTargets`, class bookkeeping.
- [Testing outlets](./outlets.md) — two controllers, one DOM, one test.
- [Mocking fetch](./fetch.md) — happy path, error path, request assertions.
- [Fake timers](./timers.md) — debounce, throttle, `setTimeout`, `setInterval`.
- [Form validation](./form-validation.md) — dispatched events, error targets, `novalidate`.
- [Keyboard & a11y](./keyboard-and-a11y.md) — `getByRole`, key grammar, `aria-expanded`, focus traps.

If you have a scenario not covered here, open an issue or a PR on [GitHub](https://github.com/tito10047/stimulus-test-utils).
